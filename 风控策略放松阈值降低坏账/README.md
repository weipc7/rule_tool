# 银行信贷风控策略系统

## 项目概述

本项目实现了一个银行信贷风控策略系统，通过多维度风险评估模型，在放宽风控规则阈值的同时，有效控制坏账率，实现审批通过率和收益的优化平衡。

## 核心功能

1. **多维度风险评估**：基于信用分数、债务收入比、月供收入比、工作年限、逾期记录、违约历史和人口统计学特征等多维度进行综合风险评估
2. **双模式风控策略**：支持严格模式和宽松模式，可根据业务需求灵活切换
3. **智能决策逻辑**：结合硬性规则检查和风险分数评估，引入补偿因素机制，实现更精准的审批决策
4. **模拟数据生成**：提供模拟用户信用数据生成功能，用于策略测试和效果分析
5. **效果分析报告**：自动生成风控策略效果分析报告，包括通过率、坏账率和风险调整后收益等关键指标

## 系统架构

### 1. 数据模型层

- `UserCreditData`：用户信用数据类，包含用户基本信息、收入、信用分数、债务情况等14个维度的特征

### 2. 风险评估层

- `RiskAssessment`：风险评估类，实现多维度风险计算
  - 信用分数风险评估
  - 债务收入比风险评估
  - 月供收入比风险评估
  - 工作年限风险评估
  - 逾期记录风险评估
  - 违约历史风险评估
  - 人口统计学特征风险评估
  - 综合风险分数计算

### 3. 风控策略层

- `RiskControlStrategy`：风控策略类，实现审批决策逻辑
  - 双模式阈值配置（严格/宽松）
  - 硬性规则检查
  - 基于风险分数的决策
  - 补偿因素机制（突破阈值的智能决策）

### 4. 测试与分析层

- 模拟数据生成
- 风控策略测试
- 效果分析与报告生成

## 使用方法

### 1. 生成模拟数据

```bash
python3 generate_sample_data.py
```

该脚本将生成包含1000条用户信用数据的CSV文件 `sample_credit_data.csv`。

### 2. 运行风控策略测试

```bash
python3 test_risk_strategy.py
```

该脚本将：
- 加载模拟数据
- 分别运行严格模式和宽松模式的风控策略
- 生成详细的效果分析报告

### 3. 直接使用风控策略

```python
from risk_control_strategy import UserCreditData, RiskControlStrategy

# 创建用户数据
user_data = UserCreditData(
    user_id="user_001",
    age=35,
    income=15000,
    credit_score=720,
    debt_to_income=0.35,
    loan_amount=100000,
    loan_term=12,
    employment_years=8,
    number_of_credit_lines=5,
    late_payments=1,
    default_history=0,
    industry="IT",
    marital_status="已婚",
    education="本科"
)

# 创建风控策略实例（宽松模式）
strategy = RiskControlStrategy(strict_mode=False)

# 评估贷款申请
evaluation_result = strategy.evaluate_loan_application(user_data)
print(f"审批结果: {evaluation_result['decision']}")
print(f"决策原因: {evaluation_result['decision_reason']}")
print(f"风险分数: {evaluation_result['risk_score']}")
```

## 策略效果对比

### 测试环境
- 模拟数据：1000条用户信用记录
- 评估指标：审批通过率、潜在坏账率、风险调整后收益

### 测试结果

| 指标 | 严格模式 | 宽松模式 | 变化 |

### 效果分析

1. **审批通过率提升**：宽松模式下通过率提升了11.90%，显著增加了贷款业务量
2. **坏账率控制**：坏账率仅增加了0.19%，增幅远小于通过率提升，体现了多维度风险评估的有效性
3. **收益优化**：风险调整后收益增加了320万元，实现了业务量和收益的双增长

## 核心算法说明

### 1. 多维度风险评估

系统采用加权平均的方式计算综合风险，各维度权重如下：

| 维度 | 权重 |
|------|------|
| 信用分数 | 35% |
| 债务收入比 | 25% |
| 月供收入比 | 15% |
| 工作年限 | 8% |
| 逾期记录 | 8% |
| 违约历史 | 5% |
| 人口统计学特征 | 4% |

### 2. 风险分数转换

采用平滑的非线性转换将加权风险转换为风险分数（40-95分），分数越高表示风险越小。

### 3. 智能决策逻辑

- **硬性规则检查**：首先检查用户是否满足基本准入条件
- **风险分数评估**：对通过硬性规则的用户进行风险分数评估
- **补偿因素机制**：对于风险分数接近阈值的用户，考虑高收入、长期工作、良好教育背景等补偿因素，实现更精准的审批决策
